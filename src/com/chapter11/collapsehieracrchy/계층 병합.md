# 계층 병합

###### Collapse Hierarchy

상위클래스와 하위클래스가 거의 다르지 않을 땐 둘을 합치자

</br>

[클래스 다이어그램]

![Collapse Hierarchy.PNG](https://raw.githubusercontent.com/machine-human/Refactoring-Study/master/image/Collapse%20Hierarchy.PNG)

</br>

[방법]

1. 상위클래스와 하위클래스 중 무엇을 삭제할지 선택

</br>

2. 필드 상향과 메서드 상향을 적용하거나

</br>

3. 메서드 하향과 필드 하향을 적용해서 삭제할 클래스의 긴능을 합칠 클래스로 전부 옮기자

</br>

4. 기능을 하나씩 옮길 때마다 컴파일과 테스트를 실시

</br>

5. 삭제할 클래스를 참조하는 부분을 합친 클래스를 참조하게 수정

</br>

6. 변수 선언, 매개변수 타입, 생성자도 이이 맞게 수정

</br>

7. 빈 클래스 삭제 후 컴파일과 테스트 실시
