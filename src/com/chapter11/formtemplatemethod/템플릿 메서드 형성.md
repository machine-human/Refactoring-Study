# 템플릿 메서드 형성

###### Form Template Method

하위클래스 안의 두 메서드가 거의 비슷한 단계들을 같은 순서로 수행할 땐

그 단계뜰을 시그너처가 같은 두 개의 메서드로 만들어서 두 원본 메서드를 같게 만든 후,

두 메서드를 상위클래스로 옮기자

</br>

[클래스 다이어그램]

![Form Template Method.PNG](https://raw.githubusercontent.com/machine-human/Refactoring-Study/master/image/Form%20Template%20Method.PNG)

</br>

[방법]

1. 추출된 메서드 전부를 똑같거나 전혀 다르게 분해

</br>

2. 메서드 상향을 실시해서 서로 같은 메서드를 상위클래스로 옮기자

</br>

3. 서로 다른 메서드를 대상으로 메서드명 변경을 실시해서 

   각 단계에서의 모든 메서드의 시그너처를 같게 만들자

   (이렇게 하면 메서드 호출 순서가 같다는 점에선 원본 메서드들이 같아지지만,)

   (아직 하위클래스/모듈의 호출 방식이 다르다)

</br>

4. 시그너처를 수정할 때마다 컴파일과 테스트를 실시

</br>

5. 원본 메서드 중 하나에 메서드 상향을 실시

   다른 메서드의 시그너처를 상위클래스에 abstract 타입의 메서드로 정의

</br>

6. 컴파일과 테스트를 실시

</br>

7. 나머지 메서드를 삭제 후 삭제할 때마다 컴파일과 테스트를 실시
