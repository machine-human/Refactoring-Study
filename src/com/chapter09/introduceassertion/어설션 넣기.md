# 어설션 넣기

###### Introduce Assertion

일부 코드가 프로그램의 어떤 상태를 전제할 땐 어설션을 넣어서 그 전제를 확실하게 코드로 작성하자

</br>

[보기]

- 변경 전

  ```java
  double getExpenseLimit() {
  	//비용 한도를 두든지, 주요 프로젝트를 정해야 한다.
  	return (expenseLimit != NULL_EXPENSE) ? 
  		expenseLimit : 	
      	primaryProject.getMemberExpenseLimit();
  }
  ```
  
- 변경 후

  ```java
  double getExpenseLimit() {
  	Assert.assertTrue(expenseLimit != NULL_EXPENSE || primaryProject != null);
  	return (expenseLimit != NULL_EXPENSE) ? 
  		expenseLimit : 	
      	primaryProject.getMemberExpenseLimit();
  }
  ```

</br>


[방법]

1. 어떤 조건이 차므로 전제된다면 어설션을 넣어 그 전제를 드러내자.
   (어설션 기능을 사요할 수 있는 Assert 클래스를 작성하자)

</br>

[예제 - 논리합(OR) 연산자]

- 수정 전

  ```java
  public class Basic {
  	double getExpenseLimit() {
  		Memeber primaryProject = new Memeber();
  		final double NULL_EXPENSE = 0;
  		double expenseLimit = 10;
  
  		return (expenseLimit != NULL_EXPENSE) ? expenseLimit : primaryProject.getMemberExpenseLimit();
  	}
  }
  
  class Memeber {
  	public double getMemberExpenseLimit() {
  		return 1.0;
  	}
  }
  ```

</br>

- 수정 후

  - Assert.assertTrue(expenseLimit != NULL_EXPENSE || primaryProject != null); 어설션 추가
  
  ```java
  public class Basic {
      double getExpenseLimit() {
          Memeber primaryProject = new Memeber();
          final double NULL_EXPENSE = 0;
          double expenseLimit = 10;
  
          Assert.assertTrue(expenseLimit != NULL_EXPENSE || primaryProject != null);
          return (expenseLimit != NULL_EXPENSE) ? expenseLimit : primaryProject.getMemberExpenseLimit();
      }
  }
  
  class Memeber {
      public double getMemberExpenseLimit() {
          return 1.0;
      }
  }
  ```
