# 매개변수 추가

###### Add Parameter

메서드가 자신을 호출한 부분의 정보를 더 많이 알아야 할 땐
객체에 그 정보를 전달할 수 있는 매개변수를 추가하자

</br>

[클래스 다이어그램]

![Add Parameter.PNG](https://raw.githubusercontent.com/machine-human/Refactoring-Study/master/image/Add%20Parameter.PNG)

</br>


[방법]

1. 메서드 시그너처가 상위클래스나 하위클래스에 구현되어 있는지 검사

   (구현되어 있다면 이 과정을 모든 구현부마다 실시)

</br>

2. 추가한 매개변수를 전달받는 새 메서드를 생성

</br>

3. 원본 메서드의 내용을 새 메서드로 복사

   (매개변수를 둘 이상 추가해야 할 때는 한꺼번에 추가하는 것이 더 간편)

</br>

4. 컴파일

</br>

5. 새 메서드를 호출하게 원본 메서드의 내용을 수정

   (참조하는 부분이 별로 없다면 이 단계를 건너뛰어도 됨)
   
   (새 매개변수에 어떤 값이든 전달해도 되지만)
   
   (보통 객체 매개변수엔 null을 전달하고)
   
   (내장 타입엔 명백히 이상한 값을 전달해서 참조하는 부분을 찾아낸다.)
   
   (쉽게 찾으려면 숫자는 0이 아닌 값을 전달하는 것이 좋다.)

</br>

6. 컴파일과 테스트

</br>

7. 원본 메서드 참조 부분을 전부 찾아서 새 메서드 참조로 바꾸자

</br>

8. 수정할 때마다 테스트를 실시

</br>

9. 원본 메서드를 삭제

   (원본 메서드가 인터페이스에 들어 있어서 삭제할 수 없다면 원본 메서드는 그대로 두고 deprecated로 표시)

</br>

10. 컴파일과 테스트
