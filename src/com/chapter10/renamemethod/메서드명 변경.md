# 메서드명 변경

###### Rename Method

메서드명을 봐도 기능을 알 수 없을 땐 메서드명을 직관적인 이름으로 바꾸자

</br>

[클래스 다이어그램]

![Rename Method.PNG](https://raw.githubusercontent.com/machine-human/Refactoring-Study/master/image/Rename%20Method.PNG)

</br>

[방법]

1. 메서드 시그너처가 상위클래스나 하위클래스에 구현되어 있는지 검사
   (구현되어 있다면 각 구현부를 대상으로 다음 단계 실시)

</br>

2. 새 이름은 새 메서드로 선언

</br>

3. 코드의 원래 내용을 새 메서드로 복사하고 적절히 수정

</br>

4. 컴파일

</br>

5. 새 메서드를 호출하게 원본 메서드의 내용을 수정
   (참조하는 부분이 적다면 이 단계는 건너뛰어도 됨)

</br>

6. 컴파일과 테스트

</br>

7. 원본 메서드 참조 부분을 전부 찾아서 새 메서드를 참조하게 수정
   (수정할 때마다 테서트 실시)

</br>

8. 원본 메서드 삭제
   (원본 메서드가 인터페이스에 들어 있어서 삭제할 수 없다면 원본 메서드는 그대로 두고 deprecated로 표시)

</br>

9. 컴파일과 테스트

</br>

[예제]

- 수정 전

  ```java
      public String getTelephoneNumber() {
          int officeNumber = 100;
          String officeAreaCode = "seoul";
          return ("(" + officeAreaCode + ") " + officeNumber);
      }
  ```

</br>

- 수정 후

  - 새 메서드 getOfficeTelephoneNumber() 메서드 명으로 작성
  - 원본 메서드 내용을 새 메서드로 복사
  - 원본 메서드 호출 부분을 새 메서드를 호출하게 수정
  - 원본 메서드 제거
  
  ```java
      @Deprecated
      public String getTelephoneNumber() {
          return getOfficeTelephoneNumber();
      }
  
      public String getOfficeTelephoneNumber() {
          int officeNumber = 100;
          String officeAreaCode = "seoul";
          return ("(" + officeAreaCode + ") " + officeNumber);
      }
  ```
